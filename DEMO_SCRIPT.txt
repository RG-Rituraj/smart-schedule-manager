[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let me walk you through the architecture. It starts with the user query passing through a security checkpoint node, which redacts emails for PII safety and checks for malicious inputs. (pause)
From there, the query is passed to the orchestrator agent, which delegates tasks to two sub-agents: the timezone resolver and the draft generator. (pause)
These sub-agents communicate with our local Model Context Protocol server to look up contact timezones, list calendar conflicts, and book events. (pause)
Finally, before sending anything, the flow routes to a human approval node which pauses the session, prompts you for validation, and only completes once you give the green light.
